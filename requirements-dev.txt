# NOT included here, on purpose: the FastSpeech2/Waveglow *training*-only stack from the legacy

#   - apex==0.9.10dev is the package that resolves to the wrong PyPI project (this is the exact

#     device = torch.device("cpu") and never imports apex).
#   - tensorflow/tensor2tensor/tgt/tensorboardX/pyworld/scikit_learn are only needed for
#     FastSpeech2's *preprocessing/training* pipeline (forced alignment, pitch/energy extraction,
#     MFA-style corpus prep), not for running synthesize.py against an already-trained checkpoint.

# numpy/torch/scipy versions below; nobody has verified that merge actually resolves.
